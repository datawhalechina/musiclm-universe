    Music Transformer STEP3 - Music Generation

    将训练好的自回归 Transformer 模型应用于推理阶段，通过给定的“音乐开头”（Prompt）或者从零开始，逐帧预测后续的 多 codebook 离散音频 token，最后还原为可听的波形文件。

    Introduction:

    1、加载模型与权重：从 checkpoint 恢复模型状态。

    2、定义生成函数：实现基于 temperature 和 multinomial 采样的自回归生成逻辑。

    3、构建 Prompt：从数据集中截取一小段音频作为“创作灵感”。

    4、音频还原：调用 Encodec 解码器将生成的 token 序列转回音频波形。

    0、环境配置与模型加载

    在生成阶段，我们需要定义与训练时完全一致的模型架构，并加载保存好的 .pt 权重文件。

    import torch
import torch.nn as nn
from music_transformer import MusicTransformer

device = "cuda" if torch.cuda.is_available() else "cpu"

# 初始化模型（参数需与训练时保持一致）
model = MusicTransformer(
    vocab_size=1024,
    num_codebooks=4,
    embed_dim=512,
    max_seq_len=8192,
    num_layers=6,
    num_heads=8,
    dropout=0.2,
).to(device)

# 加载第 4 个 Epoch 的模型权重
checkpoint = torch.load("maestro_checkpoints/checkpoint_epoch4.pt", map_location=device)
model.load_state_dict(checkpoint["model_state_dict"])
model.eval() # 切换至评估模式，关闭 Dropout

    1、构建生成逻辑（Autoregressive Inference）

    自回归生成的本质是：将当前时刻生成的 token 重新喂给模型，作为下一时刻预测的输入。

    关键参数说明： - max_new_tokens: 想要生成的总长度。 - temperature (温度): - > 1.0：让概率分布更平坦，增加随机性和创造力，但可能导致胡言乱语。 - < 1.0：让概率分布更陡峭，结果更保守、稳定。 - multinomial (多项式采样): 不是简单取概率最大的那个，而是按概率分布随机抽，这样每次生成的音乐都会有所不同。

    import torch.nn.functional as F

@torch.no_grad()
def generate(model, start_tokens, max_new_tokens=200, temperature=1.0):
    model.eval()
    x = start_tokens.to(device) # x 形状: [B, T, K]

    for _ in range(max_new_tokens):
        # 如果长度超过了模型支持的最大序列，进行裁剪
        x_cond = x[:, -model.max_seq_len:]

        # 1. 前向传播得到全量 logits [B, T, K, vocab]
        logits = model(x_cond)  

        next_tokens = []

        # 2. 针对 4 本 codebook 分别采样
        for k in range(model.num_codebooks):
            # 只取最后一个时间步 (-1) 的预测结果
            last_logits = logits[:, -1, k, :] / temperature
            probs = F.softmax(last_logits, dim=-1)

            # 按概率分布进行随机采样
            next_token = torch.multinomial(probs, num_samples=1)
            next_tokens.append(next_token)

        # 3. 将 4 本 codebook 的 token 拼回去 [B, 1, K]
        next_tokens = torch.stack(next_tokens, dim=-1)  

        # 4. 把新生成的帧追加到序列末尾，作为下一轮循环的输入
        x = torch.cat([x, next_tokens], dim=1)

    return x

    2、开始音乐生成

    我们从测试集中取一小段（比如前 10 帧，约 0.13 秒）作为引导序列（Prompt），让模型接着往下弹奏。

    from dataset import MusicTokenDataset

# 加载数据集获取样本
dataset = MusicTokenDataset(
    "maestro_tokens_3kbps",
    block_size=1024,
    stride=256
)

x_sample, _ = dataset[0]
start = x_sample[:10].unsqueeze(0)  # 截取前 10 帧作为种子

# 生成 400 个新帧 (约 5.3 秒)
generated_tokens = generate(
    model,
    start_tokens=start,
    max_new_tokens=400,
    temperature=1.1
)

# 打印一下每本 codebook 采样出的唯一 token 数量，验证多样性
for k in range(4):
    print(f"Codebook {k} unique tokens:", generated_tokens[0, :, k].unique().shape[0])

    3、音频解码与还原

    生成的是离散的数字（Token），肉耳无法听。我们需要使用 Encodec 的解码器，将这些数字转换回音频采样点（Waveform）。

    注意： Encodec 的 decode 接口接收的是一个特定格式的 list，其中包含 (codes, scale) 元组。

    from encodec import EncodecModel
import torchaudio

# 1. 加载对应的 Encodec 预训练模型
codec = EncodecModel.encodec_model_24khz()
codec.set_target_bandwidth(3.0)  # 需与训练数据时的 3kbps 一致
codec = codec.to(device)
codec.eval()

# 2. 形状变换：[T, 4] -> [4, T] 并增加 Batch 维度 -> [1, 4, T]
codes = generated_tokens[0].permute(1, 0).unsqueeze(0).to(device)

# 3. 构造 Encodec 解码格式
encoded_frames = [(codes, None)]  

# 4. 解码：Token -> Audio Waveform
with torch.no_grad():
    wav = codec.decode(encoded_frames)

    4、播放与保存结果

    最后，我们可以直接在 Notebook 中试听，或者保存为 .wav 文件。

    from IPython.display import Audio
import soundfile as sf

# 提取波形数据
audio = wav[0, 0].cpu().numpy()

# 在 Notebook 中直接播放
# Audio(audio, rate=24000)

# 保存到本地文件
sf.write("generated_music.wav", audio, 24000)
print("音乐生成完毕，已保存为 generated_music.wav")

    生成效果分析： - 结构感：如果 CB1 Loss 下降得足够好，你应该能听到具有一定节奏感和旋律起伏的钢琴声。 - 音质：音质主要取决于 Encodec 的带宽（3kbps）。如果生成的 token 符合逻辑，音质会接近原音频的压缩水平。 - 随机性：如果觉得音乐太乱，可以尝试调低 temperature 到 0.8 或 0.9。
